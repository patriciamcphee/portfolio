A few things to keep in mind when creating your pull request:

<table>
  <tr>
    <th width="40%">If</th>
    <th width="60%">Then</th>
  </tr>
  <tr>
    <td>The PR relates to issues you found in the docs</td>
    <td>add keywords such as: <p>
    <ul><li className="table-list"><b>inaccurate</b></li><li className="table-list"><b>typo</b></li><li className="table-list"><b>missing information</b></li></ul></p>
    <p className="table-paragraph">use the default base branch, <code>main</code></p></td>
  </tr>
  <tr>
    <td>The PR relates to GitHub issues or feedback you've addressed for the docs</td>
    <td>provide links to related GitHub issues and use the default base branch, <code>main</code></td>
  </tr>
  <tr>
    <td>The PR is part of the current release</td>
    <td>use the default base branch, <code>main</code></td>
  </tr>
    <tr>
    <td>The PR is part of the next future release</td>
    <td>open it against the <code>docs-staging</code> branch</td>
  </tr>
    <tr>
    <td>The PR is part of front-end or back-end development</td>
    <td>open it against the <code>active-development</code> branch</td>
  </tr>
</table>
