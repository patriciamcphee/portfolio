Design and test parameter forms in isolation. It provides a way to validate the user experience of your template forms without executing the actual template actions. 


**Perfect for:**

- Prototyping and testing parameter configurations without executing templates

- Previewing form layouts and UI components

- Validating form logic and dependencies

- Debugging conditional fields and validation rules

- Experimenting with different parameter types and UI fields

:::tip
Focus on the user experience. 

- Are field labels clear?
- Is validation helpful or frustrating?
- Does the form flow logically?
- Are error messages informative?
:::

---

1. Select **Template Form Playground** from the options.

2. Do one of the following:

   1. Paste the template's `parameters` section directly into the YAML editor to instantly preview the form.

   1. From the **Template** drop-down, select an available template for editing.

      <img
        id="uielement5"
        src={require('../images/template-form-playground.png').default}
        alt="Load Template Directory option"
      />

3. Experiment with values and input data to understand how the form behaves.

4. (Optional) Once satisfied, copy your updated YAML and paste it into your template.
