These are documentation samples written for production software — APIs, SDKs, developer platforms, and internal tooling. Each section includes context about the project, audience, and approach so you can evaluate both the writing and the thinking behind it.

## API and SDK documentation

Reference documentation written for [InsightHub](./api/index.mdx), a multi-tenant SaaS research platform built on Azure. Samples include a [CosmosDB service module](./api/cosmosdb/index.mdx) covering 60+ methods across 11 service domains, a [REST API](./api/scout-ai-rest-api.mdx) for an AI-powered research assistant, endpoint-level docs for [comments](./api/document-comments-api.mdx), [feedback](./api/feedback-api.mdx), and [invitations](./api/invitations-api.mdx), and a [TypeScript client SDK](../sdk/scout-ai-client-sdk.mdx). All were structured from source code analysis with no prior documentation to work from.

## Style guide

An [enterprise-grade style guide](./style-guide/index.mdx) built to govern documentation quality across engineering teams. Covers voice and tone, writing conventions, content types, and contributor guidelines — 25+ pages ranging from heading standards to release note formats.

## UX case studies

[Design case studies](./portfolio/index.mdx) with problem/solution/results structure, demonstrating information architecture and developer experience work on Backstage-based developer portals. Includes a sidebar navigation redesign, a template catalog redesign, and a structured prompt toolkit for AI-generated documentation.
