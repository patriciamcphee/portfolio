I've spent 15+ years making developer platforms easier to understand and use — writing API references, SDK guides, and developer documentation for teams at Microsoft, Amazon, Meta, GE Healthcare, and LivePerson. The hardest documentation problems I've encountered aren't writing problems. They're architecture, tooling, and process problems.

That realization led me to build things.

**TechWrit AI** (techwrit.ai) is a code-aware documentation platform I designed and launched in February 2026. It generates structured technical documentation from source code, configs, and specs — enforcing style rules, glossary terms, and readability standards during generation, not after. It ships with 14 documentation modes, a VS Code extension, and a REST API for CI/CD integration.

**Trellis Docs** (pixlngrid.github.io/trellis-docs) is an open-source documentation framework built on Next.js 15 that eliminates the front-end configuration overhead most teams face when setting up a production-ready docs site. It's also integrated directly into TechWrit AI as a supported output framework.

Both were built while working full-time.

At Cigna, I currently lead documentation for Alchemy, an internal developer portal built on Backstage, and own the ContentOps Playbook and enterprise style guide governing documentation quality across engineering teams. Before that, I rebuilt developer documentation at Beyond Identity, documented Azure VMware Solution with Microsoft's Azure Dedicated engineering team, and wrote SDK and API guides for conversational AI platforms at LivePerson.

My contract at Cigna ends in June 2026. If you're building a developer experience function, scaling a documentation team, or need someone who can own documentation strategy end to end — including the tooling — I'd like to talk.

>**What I'm looking for:** Technical Content Manager and Developer Experience roles, remote.
